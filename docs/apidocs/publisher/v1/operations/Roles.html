roles

Validate whether the logged-in user has the given role

HEAD https://apis.wso2.com/api/am/publisher/v1.0/me/roles/{roleId}

Using this operation, logged-in user can check whether he has given role.

OAuth 2.0 Scope

apim:api_create

Examples

Parameters

Type Name Description Schema Default
Path

roleId

required

The Base 64 URL encoded role name with domain. If the given role is in secondary user-store, role ID should be derived as Base64URLEncode({user-store-name}/{role-name}). If the given role is in PRIMARY user-store, role ID can be derived as Base64URLEncode(role-name)

String

Responses

HTTP Code Description Schema

200

OK. Requested user has the role.

404

Not Found. Requested user does not have the role.
